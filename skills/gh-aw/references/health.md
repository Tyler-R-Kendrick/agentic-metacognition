# gh aw health

Show health metrics and success-rate trends for workflows.

## When to use it

Use when you need an operational view of workflow reliability over time.

## How to use it

1. Start with `gh aw health --help` to see the flags for your installed version.
2. Run the smallest command that answers your immediate need.
3. Review the diff or GitHub-side effect before moving on when the command changes repository state.

## Example commands

- `gh aw health`
- `gh aw health daily-gh-aw-training`

## Notes

- Combine this with `status`, `logs`, and `audit` for deeper diagnosis.

## Verify

- Re-run `gh aw health --help` if you are unsure about supported flags in the installed release.
